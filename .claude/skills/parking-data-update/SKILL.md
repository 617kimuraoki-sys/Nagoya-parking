# 駐車場料金データ更新スキル

## 概要
`data.js` に記載された駐車場の料金を、Google マップおよびタイムズ公式サイトで確認し、変更があれば更新する。

---

## 実行前の準備

### 1. 対象を確認する
ユーザーの指示から対象範囲を判断する：

| 指示の例 | 対象 |
|---------|------|
| 「全件確認して」 | 全107件 |
| 「栄エリアを更新して」 | area: "栄" のみ |
| 「10件だけ試しに」 | 先頭10件 |
| 「タイムズだけ」 | name に「タイムズ」を含む |

### 2. data.js から対象データを読み込む
```
ファイル: /Users/k25040kk/nagoya-parking/data.js
```
各駐車場は以下の構造を持つ：
```javascript
{
  name: "駐車場名",
  area: "栄",
  address: "愛知県名古屋市...",
  rate: {
    weekday: "平日料金テキスト",
    holiday: "休日料金テキスト",
    weekdayMax: 800,      // 円
    holidayMax: 900,      // 円
    per30min: 50,         // 試算用
    per30minHoliday: 100,
  },
  note: "補足（任意）"
}
```

---

## 料金確認の手順（1件ずつ実行）

### Step 1: Google マップで検索
- 検索キーワード: `{name} {address}`
- Google マップの駐車場詳細ページで料金情報を確認する
- 料金が画像・テキストで表示されていれば読み取る

### Step 2: 料金が見つからない場合
以下の順で追加確認する：

**タイムズの場合** → タイムズ公式サイトで検索
- URL: `https://www.timesclub.jp/times_p/search/`
- 駐車場名または住所で検索して料金ページを確認

**その他ブランドの場合** → スキップして「不明」として記録

### Step 3: 料金を比較する
取得した最新料金と `data.js` の現在データを比較：
- 平日料金（テキスト・最大料金）
- 休日料金（テキスト・最大料金）
- 基本単価（per30min）

---

## 結果の記録

確認ごとに以下の3つに分類する：

| 分類 | 内容 |
|------|------|
| ✅ 変更あり | 料金差異を検出 → data.js を更新予定 |
| ⬜ 変更なし | 現在データと一致 |
| ❓ 不明 | Google マップ・公式サイトで料金確認できず |

---

## 完了後：レポートを出力する

全件確認が終わったら、以下の形式でレポートを表示する：

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📋 駐車場料金更新レポート
対象: {エリア名 or 全件} / {件数}件確認
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✅ 変更あり: {N}件
  - {駐車場名}
    現在: 平日最大 800円 → 最新: 1,000円
  - ...

⬜ 変更なし: {N}件

❓ 料金不明: {N}件
  - {駐車場名}（Google マップ・公式サイトで未確認）
  - ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
変更を data.js に反映して push しますか？
```

---

## push の手順（ユーザー確認後）

ユーザーが「はい」「push して」などと返答した場合のみ実行：

```bash
cd /Users/k25040kk/nagoya-parking
git add data.js
git commit -m "fix: 駐車場料金データを最新情報に更新（{エリア}/{N}件）"
git push origin main
```

**ユーザーが確認する前に push しないこと。**

---

## 注意事項
- Google マップの料金表示は画像・テキスト混在のため、読み取れない場合は「不明」として記録する
- 料金変更の確信が持てない場合は「不明」扱いにして、ユーザーに判断を委ねる
- per30min（試算用単価）は料金テキストから推計して更新する
- 1件あたり約1〜2分かかるため、全件実行は時間がかかる旨を事前に伝える
